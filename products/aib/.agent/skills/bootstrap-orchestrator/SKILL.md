# Bootstrap Orchestrator

Use this skill when a user brings a vague product idea and wants a project scaffold that can move from idea to implementation.

Outcomes:

- discovery interview in small batches
- accepted `docs/spec.md`
- milestone docs under `docs/milestones/`
- issue drafts or GitHub issues
- projected agent-tool assets built from `.agent/`

Key files:

- `.qube/aib/session.json`
- `.qube/aib/discovery-log.md`
- `.qube/aib/assumptions.md`
- `docs/spec.md`
- `docs/milestones/`
- `docs/issues/`
- `AGENTS.md`

Core rules:

- discovery before architecture detail
- invariants before implementation detail
- explicit acceptance before milestones
- profile-aware issues and verification
- `.agent/` is the source of truth for reusable agent assets
