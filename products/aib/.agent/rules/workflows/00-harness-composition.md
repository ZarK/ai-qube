# Harness Composition

- Compose the final project harness from reusable fragments instead of copying one project's workflow whole.
- Start with a small, portable core and add stricter profile rules only when the project actually needs them.
- Prefer best-effort projections for non-primary tools instead of inventing unsupported native integrations.
- If a tool already converges on `AGENTS.md`, use that as the shared baseline and add mirrored files like `CLAUDE.md` or `GEMINI.md` only as convenience entrypoints.
