QUBE Command Surface: Idea To Complete Implementation

Use QUBE as the entrypoint and dispatcher. AIB owns planning, AIE owns issue and PR execution, AIQ owns quality evidence, and AIU owns continuation policy.

QUBE AIB AIE AIQ AIU Current handoff gap

1. Discover

QUBE

qube make-it-so "..." qube components qube run <tool> -- --help

Confirms installed component versions and the available command family.

2. Capture Idea

AIB

qube run aib -- init . --agent codex --idea "..." --json qube run aib -- next --json qube run aib -- answer --field <field> --value <answer> --json

Turns the fuzzy request into structured planning state.

3. Accept Spec

AIB

qube run aib -- spec draft --json qube run aib -- spec validate --json qube run aib -- spec accept --section <id|all> --json

Implementation should not start until the right sections are accepted.

4. Work Items

AIB

qube run aib -- milestones generate --json qube run aib -- work-items generate --json qube run aib -- work-items render --provider github --dry-run --json

Produces provider-ready work item drafts.

5. Issue Handoff

Boundary

review rendered GitHub issue draft create or confirm real GitHub issue

Current gap: AIB renders issue output; AIE starts from real provider issues.

6. Execute Issue

AIE

qube run aie -- queue --json qube run aie -- start next qube run aie -- view <issue> qube run aie -- branch check <issue>

Selects one valid issue and moves work onto the issue branch.

7. Implement + Audit

AIE

edit code, tests, and docs qube run aie -- audit ui <issue> --prepare qube run aie -- run start --name ui-audit -- <dev command> qube run aie -- run wait --name ui-audit --url <url> --timeout 30 qube run aie -- run stop --name ui-audit

Uses managed local app runners for UI evidence when needed.

8. Quality Evidence

AIQ

qube run aiq -- setup --format json qube run aiq -- doctor --format json qube run aiq -- plan . --format json qube run aiq -- run . --format json qube run aiq -- evidence --format json

Produces trusted machine-readable gate evidence.

9. PR To Completion

AIE + Git

qube run aie -- pr body <issue> git commit && git push open non-draft PR that closes issue qube run aie -- pr view <pr> --json qube run aie -- pr gate <pr> --json merge after approval, CI, and gates qube run aie -- complete <issue>

PR helpers do not silently merge. Merge happens only after policy passes.

Failure Loop

If AIQ, CI, review, or UI audit fails, return to implementation, fix the concrete issue, rerun the affected gate, then rerun qube run aie -- pr gate <pr> --json.

Continuation Loop

After merge and aie complete, AIU can inspect trusted state with qube run aiu -- status --json, host hooks, or whip tasks before the next issue starts.
