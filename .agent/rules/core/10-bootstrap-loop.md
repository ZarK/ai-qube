# Bootstrap Loop

The bootstrap system follows this order:

1. idea
2. discovery interview
3. dry spec draft
4. spec revision loop
5. accepted spec
6. milestones
7. issues
8. harness and tool projection

Operating rules:

- discovery should happen in small batches, not giant questionnaires
- specs should be invariants-first, contract-oriented, and acceptance-hooked
- milestones start only after the spec is accepted
- issue generation starts only after milestones exist
- project harnesses should be composed from the chosen profile, not copied blindly from one existing project
